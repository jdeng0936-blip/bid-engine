## 一、公司概况

安徽徽远成电气科技股份有限公司（简称"徽远成"）成立于2010年，总部位于安徽省合肥市，是一家深耕电气安全与工业智能化领域的科技型企业。历经十五年发展，公司已形成**高压电气监测保护设备研发制造**与**工业AI软件研发与应用赋能**两大核心业务板块，拥有专业技术团队77人，构建起"硬件感知 + 软件智能 + AI大脑"三位一体的综合服务能力体系。

公司坚持以自主创新为驱动，长期专注于高压电气监测保护类产品的研发、生产与销售，同时积极布局人工智能技术在工业领域的落地应用，为煤矿、能源、电力、制造等行业客户提供从设备层到数据层再到智能决策层的全栈式解决方案。

## 二、发展历史沿革

> 从电气硬件制造商，到工业数据平台服务商，再到行业AI赋能引领者——徽远成用十五年走出了一条"硬件筑基、数据蓄力、AI跃迁"的进化之路。

### 2010—2014  |  深耕电气，锻造工业基因

公司于2010年在合肥高新区创业起步，创始团队来自电力系统与自动化控制领域，从第一天起就将研发能力锚定在高压电气监测保护这一高技术壁垒领域。这五年间，团队从零完成了智能终端、无线测温两大温度监测产品线的自主研发与定型；攻克了动态无功补偿、有源滤波、智能无功补偿电容器三大电能质量核心产品；打造了无线自诊断过电压保护装置、智能消弧消谐装置等八大用电安全类设备，并取得多项国家发明专利。

**里程碑：** 产品开始批量进入国家电网、南方电网、中石油、中石化及五大发电集团，建立起面向大型央企的直接供货体系。公司在这一阶段积累的最核心资产，不是设备本身，而是**对工业现场高压电气系统运行规律的深层理解**——这正是后来AI赋能工业场景时，其他纯软件公司难以复制的竞争壁垒。

### 2015—2018  |  股改挂牌，软硬一体化战略升级

2015年7月，公司在全国中小企业股份转让系统成功挂牌（股票代码：**832943**），标志着徽远成从技术型创业企业迈入规范化资本运营阶段。同年，公司在深圳成立研究院，引入211高校研究生和博士级高端人才，高科技人才占研发团队70%以上。

这一阶段的战略关键词是**"从卖设备到卖数据服务"**。公司组建了专业软件研发团队，自主开发了工业数据采集与监控（SCADA）平台，首次实现了设备出厂后的远程运行状态监测：温度、电流、振动等关键参数通过无线传输实时汇聚至云端，客户可以在办公室里查看数百公里外变电站设备的健康状态。

**能力跃迁：** 从"交付硬件产品"升级为"交付硬件+数据平台"一体化解决方案，初步构建起"设备感知层→数据汇聚层→可视化展示层"的三层技术架构——这一架构在五年后，自然演化为AI赋能的底层骨架。

### 2019—2021  |  数据蓄力，构建AI就绪的数据底座

工业互联网浪潮兴起，公司敏锐地意识到：**数据是AI时代最稀缺的生产资料，而我们已经守着金矿却浑然不知。** 这一阶段，公司启动了全面的数字化转型：

- **协议层突破：** 完成MQTT、Modbus TCP/RTU、OPC-UA等主流工业协议的适配开发，打通了PLC、DCS、RTU等异构设备的数据壁垒
- **数据中台构建：** 自研工业数据清洗与特征提取流水线，实现时序数据的毫秒级采集、分钟级聚合与结构化存储
- **行业数据积淀：** 先后承接矿山、变电站、生产车间等多个数字化改造项目，积累了覆盖高压开关柜、变压器、电缆测温等十余类设备的海量运行数据

**关键积累：** 公司手中不再只有硬件产品，而是拥有了**来自真实工业现场、经过清洗标注的高质量运行数据**。这些数据为后续训练设备异常检测模型、构建行业知识库提供了不可替代的素材——事实证明，这是AI技术从"实验室概念"走向"工业落地"最难逾越的一道门槛。

### 2022—2023  |  拥抱大模型，AI赋能战略正式启航

2022年末，ChatGPT引爆全球AI浪潮。国内的文心一言、通义千问、DeepSeek等大语言模型相继发布。公司管理层敏锐判断：**大模型将深刻改变工业软件的交互方式和知识处理范式**，必须抢占先机。

公司于2022年底正式成立AI应用研发小组（4人核心团队），围绕三条主线展开系统性技术攻关：

1. **大模型工程化落地：** 系统评估了GPT-4、DeepSeek-V2、通义千问Qwen-Max等主流模型在工业文档理解、专业计算辅助、合规性判断等场景的表现，最终确立"国产模型优先+API调用为主+本地部署预研"的技术路线
2. **RAG检索增强生成：** 攻克了工业文档的智能解析（PDF/Word→结构化切片→语义向量化→pgvector存储→HNSW索引检索）全链路工程化，实现了万级知识片段的毫秒级语义检索
3. **工程计算引擎融合：** 创新性地将传统工程计算公式（赫尔曼支护公式、通风需风量三法计算等）以Tool Calling的方式注入大模型推理链，使AI不仅能"写文字"，还能"算数据"

**验证突破：** 2023年第四季度，首个AI赋能产品原型——"招投标标书智能生成引擎"通过内部验证测试，在真实招标文件上实现了技术标全文生成，质量评分达到资深工程师手工编写水平的85%以上，验证了AI技术在专业文档智能生成领域的可行性与商业价值。

### 2024—2025  |  深度垂直，AI赋能规模化落地

此阶段，公司将**"AI应用赋能"正式确立为第二增长曲线**，在保持硬件主业稳健经营的基础上，集中研发资源深耕工业垂直大模型应用：

**产品一：招投标标书AI智能生成平台**
面向中铁四局电气化局等大型施工企业，构建了涵盖30余份历史中标标书、20余项行业规范的私有知识库（3,000+语义片段）。平台支持商务标、技术标、资信标三类标书的一键生成，编制效率提升75%以上，合规漏项率下降90%。

**产品二：高压电气设备AI智能运维平台**
基于十五年设备数据积淀，构建了LSTM+规则引擎混合异常检测模型，故障预警准确率＞95%。同时部署RAG驱动的AI运维问答助手，一线工程师用自然语言即可查询设备手册和故障处置规程。

**产品三：掘进工作面规程AI智能生成平台**
面向华阳集团旗下矿井，构建了包含2,005条语义片段的矿山行业私有知识库。内置三套专业工程计算引擎，实现了11章节完整规程的AI智能生成与合规校验，规程编制周期从5-15天缩短至30分钟以内，效率提升80%以上。

**能力定型：** 至此，徽远成已形成完整的**"行业知识库构建→RAG语义检索→工程计算引擎→大模型生成→合规校验→反馈迭代"全链路AI赋能交付能力**，具备向煤矿、电力、建筑等多行业复制推广的技术成熟度。

## 三、AI 赋能技术积累

### 3.1 技术体系总览

徽远成的AI赋能能力以"三层架构"为核心：

**感知层（硬件根基）**
凭借十五年高压电气监测设备的研发经验，公司在工业数据采集领域具备深厚积累，能够从PLC、DCS、传感器等各类工业设备获取高质量的实时数据。这一能力是AI模型训练和推理的可靠数据来源。

**平台层（数据中台）**
公司自主研发了工业数据处理平台，支持MQTT、Modbus、OPC-UA等主流工业协议的接入与解析，具备数据清洗、特征提取、实时流处理能力，可为AI应用提供高质量的结构化数据输入。

**智能层（AI大脑）**
基于国产大语言模型（DeepSeek、通义千问）和RAG（检索增强生成）技术，公司已形成完整的工业AI应用开发能力，涵盖：行业知识库构建与向量化、工程计算引擎集成、LLM Tool Calling工具调用、AI内容生成与合规校验。

### 3.3 典型AI项目案例

**案例一：招投标标书AI智能生成平台（2024-2025）**

服务对象：中铁四局电气化局
主要成果：
- 构建招标文件私有知识库：收录历史中标标书30余份、行业技术规范20余项，共计3,000余条语义片段，完成向量化入库
- 开发AI标书生成引擎：支持商务标、技术标、资信标三大类型，全文生成时间＜5分钟，效率较手工编制提升75%以上
- 部署标书合规自动校验模块：实现页码完整性、章节结构、资质要求、投标报价一致性自动核验，漏项率下降90%
- 接入AI智能问答助手：自然语言检索历史标书、条目匹配与规范引用，响应时间＜3秒
- 集成投标报价分析模块：基于历史中标数据辅助生成合理报价区间，中标概率显著提升

**技术链路：**

```
招标文件（PDF/Word）解析 → 语义切片 → BGE-M3 向量化 → pgvector 存储
            ↓
用户录入标书关键参数（项目类型 / 规模 / 工期 / 资质要求）
            ↓
RAG 检索（相似历史标书 Top5 + 行业规范 Top3）→ 注入 Prompt
            ↓
DeepSeek-V3 生成技术方案 / 商务条款 / 项目实施计划（SSE 流式输出）
            ↓
Jinja2 模板渲染 → Word 文档导出（标准封面 + 自动目录 + 正文）
            ↓
合规自动校验 → 问题清单高亮 → 人工审核确认 → 正式投标
```
**案例二：高压电气设备AI智能运维平台（2023-2024）**

主要成果：
- 构建设备知识库：收录30余类高压电气设备的操作手册、故障代码库、维保规程，向量化入库，共计5,000余条知识片段
- 开发设备异常检测模型：基于历史传感器时序数据（电流/温度/振动），构建LSTM+规则引擎混合检测模型，故障预警准确率＞95%，误报率＜5%
- 部署AI运维问答助手：一线工程师用自然语言查询故障代码、处置规程，平均响应时间＜3秒，替代翻阅纸质手册
- 建立预测性维护体系：结合设备运行曲线异常分析，提前3-7天预警潜在故障，累计协助客户避免非计划停机数十次
- 实现移动端接入：支持平板/手机端实时查询，适配井下弱网环境的离线知识缓存

**技术链路：**

```
设备传感器数据采集（MQTT/Modbus） → 实时流处理（时序清洗/特征提取）
            ↓
LSTM 时序异常检测模型 → 规则引擎（阈值/趋势）→ 预警等级分级
            ↓
设备手册/故障代码 PDF 解析 → 语义切片 → BGE-M3 向量化 → pgvector 存储
            ↓
用户自然语言提问 → RAG 检索（相关文档 Top5）→ 注入 Prompt
            ↓
通义千问 Qwen-Max 生成处置建议（流式输出）
            ↓
Web/移动端展示（含设备档案卡、故障历史、维保计划）
```

```

            ↓
DeepSeek-V3 生成11章节规程文本（SSE 实时流式输出）
```
