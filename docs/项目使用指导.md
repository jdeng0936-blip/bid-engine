# 鲜标智投 — 项目使用指导

## 一、平台定位

**鲜标智投**是一个 AI 驱动的**生鲜食材配送投标文档智能生成平台**，目标是将投标文档准备从 3-5 天缩短到 30 分钟。

---

## 二、投标项目 (BidProject) 说明

系统中的项目列表对应的是**投标项目 (BidProject)**。每个项目代表一次对某个招标的投标响应。

| 字段 | 含义 |
|------|------|
| 项目名称 | 如"测试学校食材配送项目" |
| 状态"草稿" | 项目处于初始阶段，尚未开始生成投标文档 |
| 金额(如 50.0 万元) | 该招标的预算金额 |
| 客户类型(学校食堂/医院等) | 招标方的类型，影响文档模板和内容策略 |

---

## 三、项目完整生命周期（核心流程）

```
草稿(DRAFT) → 解析中(PARSING) → 已解析(PARSED) → 生成中(GENERATING)
→ 已生成(GENERATED) → 审核中(REVIEWING) → 已完成(COMPLETED)
→ 已提交(SUBMITTED) → 中标(WON) / 未中标(LOST)
```

### 操作流程

1. **创建项目** — 填写项目名称、招标方信息、预算金额、客户类型（学校/医院/政府/企业/食堂）、招标类型（公开/邀请/竞争性谈判等）
2. **上传招标文件** — 上传招标方的 PDF/Word 文件
3. **智能解析** — AI 自动从招标文件中提取：
   - 废标项（最关键，触发即出局）
   - 资格要求
   - 技术要求
   - 商务要求
   - 评分标准
4. **初始化报价** — 系统根据招标要求自动生成报价单，按 6 大食材品类预填常见品项和市场参考价
5. **生成投标章节** — AI 结合知识库（法规/模板/历史中标案例）自动生成 9 个标准章节
6. **合规检查** — 5 维度自动检查（废标项/资格/评分覆盖/技术/商务）
7. **导出 Word** — 一键导出包含封面、目录、章节、报价表、资质清单的完整投标文档

## 四、核心功能模块说明

### 1. AI 智能对话

- 支持自然语言提问，如"冷链运输有什么法规要求？"
- AI 自动路由到 6 个专业工具：
  - **法规检索** — 语义搜索食品安全法律、冷链标准、采购规定
  - **资质验证** — 匹配招标要求与企业资质
  - **章节生成** — AI 结合 RAG 生成投标文档章节
  - **定价分析** — 基于历史中标数据建议报价策略
  - **合规检查** — 5 维度合规审查
  - **案例搜索** — 检索相似历史中标案例
- 支持多轮对话，历史记录持久化

### 2. 报价管理

- **6 大食材品类**：蔬菜、肉类、水产、蛋禽、干货、调料
- 支持三种定价方式：固定价格、下浮率、综合报价
- 下浮率计算公式：`单价 = 市场参考价 × (1 - 下浮率)`
- 从招标要求自动初始化报价单，预填常见品项和市场参考价

### 3. 企业资质管理

- 管理 15 类资质证书：
  - 食品经营许可证、营业执照、HACCP、ISO22000、SC 认证
  - 动物检疫证、冷链运输资质、健康证、责任保险
  - 质量检测报告、有机认证、绿色食品认证、业绩证明、获奖证书
- 追踪冷链资产（冷藏车数量、仓库面积、冷库温度范围）
- 系统自动匹配招标要求，识别缺失资质

### 4. 知识库

- **法规标准库** — 食品安全法、冷链标准、采购规定等
- **章节模板库** — 按客户类型区分的可复用模板片段（Jinja2 模板）
- **历史中标案例** — 用于定价策略参考和相似项目推荐
- 全部支持语义向量检索（pgvector 1536 维向量嵌入）

### 5. 合规检查引擎（5 维度）

| 维度 | 说明 | 严重程度 |
|------|------|----------|
| 废标项 | 触发即出局的硬性条件 | 最高 |
| 资格要求 | 企业资质是否齐全 | 高 |
| 评分覆盖 | 章节是否覆盖所有评分点 | 中 |
| 技术要求 | 关键技术指标是否响应 | 中 |
| 商务条款 | 商务条件是否满足 | 一般 |

**验证方式**：
- 第一层：规则引擎（关键词匹配、资质匹配）— 快速确定性检查
- 第二层：LLM 语义审查 — 处理边缘情况的精确检查

## 五、客户类型与应用场景

| 客户类型 | 典型场景 |
|----------|---------|
| 学校食堂(school) | 学校食材配送招标，如"XX市第一中学" |
| 医院(hospital) | 医院食堂食材供应 |
| 政府(government) | 政府机关食堂采购 |
| 企业(enterprise) | 企业团餐食材供应 |
| 食堂(canteen) | 通用食堂类型 |

不同客户类型会影响：文档模板选择、章节内容侧重、合规检查规则。

## 六、招标类型

| 类型 | 说明 |
|------|------|
| 公开招标(open) | 面向所有符合条件的供应商 |
| 邀请招标(invite) | 向特定供应商发出邀请 |
| 竞争性谈判(negotiate) | 与多家供应商谈判 |
| 询价(inquiry) | 向供应商询价比较 |
| 单一来源(single) | 仅向一家供应商采购 |

## 七、投标文档章节结构（9 个标准章节）

AI 自动生成的投标文档包含 9 个标准章节，每个章节内容来源可以是：
- **template** — 基于模板生成
- **ai** — AI 智能生成
- **manual** — 用户手动编写
- **credential** — 基于企业资质自动填充

每个章节有独立状态：草稿(draft) → 已生成(generated) → 已审核(reviewed) → 已定稿(finalized)

## 八、系统架构要点

### 多租户隔离
- 每个注册用户自动创建租户，数据完全隔离
- 所有查询强制校验 `tenant_id`
- RAG 检索按租户过滤

### 用量配额
- 项目数、导出次数、AI 调用次数有上限
- 通过 `user_quota` 和 `usage_log` 表管理

### 用户反馈闭环
- AI 生成内容支持"采纳/编辑/拒绝"三种操作
- 记录原始文本、修改后文本、编辑距离
- 数据用于未来 SFT（监督微调）持续优化模型

### 文档导出格式
- 封面页
- 目录
- 9 个投标章节（支持 Markdown 转 Word 格式化）
- 报价表（按品类分表）
- 资质清单
- 嵌入的图片资源

## 十一、安全与设计原则

1. **零信任密钥** — LLM API Key 仅存于后端 `.env`，绝不暴露给前端
2. **优雅降级** — LLM 请求超时处理 + 缓存回退
3. **安全护栏** — 报价、合规等安全关键值由规则引擎计算，不盲信 LLM 输出
4. **多租户隔离** — 所有查询必须校验 `tenant_id`
5. **SSE 流式输出** — AI 响应使用 Server-Sent Events 实时推送
6. **二进制零数据库** — 文件存 OSS/S3，数据库只存 URL 和可检索文本
7. **全异步架构** — FastAPI 所有 I/O 操作使用 `async def`
8. **Pydantic V2 校验** — 所有 API 输入严格校验
9. **反馈闭环** — 每个 AI 输出必须有采纳/编辑/拒绝 UI
10. **LLM Tool Calling 路由** — 业务逻辑通过 LLM 意图路由，不硬编码 if-else
