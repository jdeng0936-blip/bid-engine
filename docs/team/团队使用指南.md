# Antigravity 团队使用指南

> 本项目使用 **Antigravity** (Google Deepmind) 作为 AI 编程助手。
> 团队成员按以下步骤配置后，AI 助手会自动遵循本项目的所有架构规范和约束。

---

## 📥 第一步：安装 Antigravity

从官方渠道安装 Antigravity，安装完成后即可在 VSCode 中使用。

---

## ⚙️ 第二步：配置全局规则（GEMINI.md）

全局规则让 Antigravity 在**所有项目**中都遵循团队技术基线。

```bash
# macOS 路径
~/.gemini/GEMINI.md
```

**操作步骤**：

```bash
# 1. 创建目录（如果不存在）
mkdir -p ~/.gemini

# 2. 复制团队模板
cp /path/to/project/docs/team/GEMINI-template.md ~/.gemini/GEMINI.md

# 3. 验证配置
cat ~/.gemini/GEMINI.md
```

> ⚠️ 如果你已经有 `GEMINI.md`，将 `GEMINI-template.md` 的内容**追加**到已有文件末尾，不要覆盖。

---

## 📂 第三步：了解项目规则文件

本项目在 `.agents/` 目录下有项目级规则文件，**Antigravity 会自动加载**，无需手动操作。

```
煤炭/
├── .agents/
│   ├── 01-AI-Platform-Rules.md    # 核心架构约束 + 红线
│   └── 02-Backend-Tech-Rules.md   # 技术栈规范 + 代码模式
├── docs/
│   └── team/
│       ├── GEMINI-template.md     # 全局规则模板（本文件）
│       └── 团队使用指南.md         # 本文档
└── ...
```

---

## 🚀 第四步：开始工作

### 打开项目
```bash
# 在 VSCode 中打开项目根目录（重要！必须是项目根目录）
code /path/to/煤炭
```

### 验证规则已加载
在 Antigravity 对话框中输入：
```
你现在遵循哪些约束规则？
```
AI 应该能列出本项目的红线约束（禁止硬编码模型名、RAG三层过滤、tenant_id隔离等）。

### 日常开发示例对话

**添加新功能**：
```
按照项目契约规范，帮我在 doc_generator.py 中添加"目录自动生成"功能，
要求通过 Critic 打分，Prompt 放入注册表中
```

**调试问题**：
```
ai_router.py 的 Tool Calling 没有调用 calc_support 工具，帮我分析原因
```

**切换 AI 模型**：
```
帮我把 doc_section_generate 任务的模型切换为 gemini-2.5-pro，
要在正确的地方改
```

## 📋 团队规范速查

### 文件不能入库 Git
```
backend/.env              ← API Key（每人本地自己配）
backend/storage/outputs/  ← 生成的 Word 文档
backups/*.dump             ← 数据库备份
*.bak                      ← 临时备份文件
```

### 必须修改这里而不是代码
| 要做的事 | 修改位置 | 禁止操作 |
|---------|---------|---------|
| 切换 AI 模型 | `llm_registry.yaml` | 改业务代码里的模型名 |
| 修改 Prompt | `prompts_registry.yaml` | 在 .py 文件里写 Prompt |
| 添加新工具 | `ai_router.py` 的 `TOOLS` 列表 | 用 if-else 判断意图 |
| 调整合规规则 | `compliance_engine.py` | 在 Prompt 里描述规则 |

### Git 提交格式
```bash
git commit -m "feat: 添加目录自动生成功能"
git commit -m "fix: 修复 Critic 打分解析异常"
git commit -m "docs: 更新立项研究文档技术描述"
git commit -m "refactor: 重构 RAG 检索权重计算"
```

## 🐛 常见问题

**Q: Antigravity 没有按项目规范回答怎么办？**  
A: 确认用 VSCode 打开的是项目**根目录**（不是子目录），`.agents/` 文件才会被自动识别。

**Q: AI 给出了硬编码模型名的代码怎么办？**  
A: 直接告诉 AI："这违反了项目红线，模型名必须从 llm_registry.yaml 读取，请修正"。

**Q: 如何知道自己的代码改了哪些地方？**  
A: 使用 `git diff` 查看变更，或在 Antigravity 对话中说"帮我 review 这次的改动是否符合约束"。

**Q: 新接入项目，知识库是空的怎么办？**  
A: 运行 `python scripts/ingest_standards.py` 导入知识库文档，再运行 `embed_standards.py` 做向量化。
